==================================================
第一阶段：可分析性核查与基础样例
==================================================

一、目标
确认现有数据库是否支持“人工调度是否复用前一个同类日”的分析。
同类日定义：
- NORMAL 对比前一个 NORMAL
- COLD_ONLY 对比前一个 COLD_ONLY

二、字段核查结果
1) human_dispatch_routes（可作为主分析明细）
- 有：delivery_date, route_id, store_id, stop_sequence, vehicle_id, driver_name
- 有货量字段：
  ambient_turnover_boxes, ambient_cartons, ambient_full_cases,
  frozen_bags, frozen_cartons, cold_bins, cold_cartons
- 无：day_type, driver_id, route_load（可由明细汇总计算）

2) chaos_replay_tasks（可提供 day_type）
- 有：delivery_date, day_type, total_load_w1, total_load_w2, total_load
- 无：store_id, route_id, stop_sequence

3) chaos_replay_task_stores（店级汇总可辅助）
- 有：delivery_date, store_id, sum_rpcs/sum_rcase/sum_rpaper/sum_bpcs/sum_bpaper/sum_apcs/sum_apaper, load_w1, load_w2
- 无：route_id, stop_sequence, day_type

4) rengong_route_distance_load_profiles（线路画像可辅助）
- 有：delivery_date, route_id, route_load, normal_load_limit, route_group, route_type, is_city
- 无：store_id, stop_sequence, day_type

结论：
- 可以判断“店铺集合复用”和“顺序复用”（因为有 stop_sequence）。
- 不能做严格司机归因（无 driver_id，仅可展示 driver_name）。

三、第一阶段样例与总体结果
1) 复用类型总体计数（prev 同类日存在条件下）
- EXACT_COPY：5151
- PARTIAL_CHANGE：982
- UNKNOWN_CHANGE：672
- SAME_STORES_DIFF_ORDER：298
- NEW_ROUTE：223
- MISSING_PREV_ROUTE：84

2) route_id 稳定性
- NORMAL：同 route_id 在前一同类日仍存在比例 97.03%
- COLD_ONLY：比例 96.21%

3) 样例观察
- NORMAL 的 2025-11-10 对比 2025-11-08：大量线路出现 PARTIAL_CHANGE/UNKNOWN_CHANGE
- COLD_ONLY 的 2025-11-16 对比 2025-11-09：多条线路 EXACT_COPY，结构复用明显

第一阶段结论：
- “同类日 route_id 高稳定复用”成立。
- 周日（COLD_ONLY）结构复用更强。

==================================================
第二阶段：变化类型拆解与货量触发关系
==================================================

一、目标
解释 PARTIAL_CHANGE / UNKNOWN_CHANGE / SAME_STORES_DIFF_ORDER 的发生原因，
并判断是否主要由货量触发。

二、变化原因拆解（第二阶段标签）
- LOAD_DRIVEN_CHANGE
- STRUCTURE_SPLIT
- STRUCTURE_MERGE
- ROUTE_REASSIGN
- ORDER_ONLY_CHANGE
- ZERO_OR_TINY_LOAD_ROUTE
- DATA_ISSUE
- UNKNOWN_REASON

三、关键统计结果
1) |route_load_delta| 分布对比
- EXACT_COPY：avg_abs_delta=0.113090, p50=0.0650, p90=0.2603
- PARTIAL_CHANGE：avg_abs_delta=0.184899, p50=0.1312, p90=0.4089
- UNKNOWN_CHANGE：avg_abs_delta=0.215895, p50=0.1607, p90=0.4810
- SAME_STORES_DIFF_ORDER：avg_abs_delta=0.145339, p50=0.1020, p90=0.2886

解释：变化类（尤其 UNKNOWN/PARTIAL）伴随更大货量波动。

2) route_load 分位数（全量观测）
- q10=0.1855, q25=0.3030, q50=0.4439, q75=0.6703, q90=0.9910

3) 按负载分层的变化率（变化=PARTIAL+UNKNOWN+ORDER）
- zero_or_tiny：23.19%
- low：30.91%
- medium：23.87%
- high：31.00%

解释：低段与高段变化率高于中段。

4) 按 load_limit_ratio 带宽（仅有上限口径的线路）
- <0.8：变化率 26.66%
- 0.8-0.9：30.26%
- 0.9-1.0：23.53%
- >=1.0：54.84%

解释：超过上限时变化率显著升高。

四、UNKNOWN_CHANGE 拆解（必须项）
UNKNOWN_CHANGE 总数：672
拆解结果：
- STRUCTURE_SPLIT：614
- STRUCTURE_MERGE：8
- ROUTE_REASSIGN：10
- ZERO_OR_TINY_LOAD_ROUTE：39
- LOAD_DRIVEN_CHANGE：0（在该严格口径下）
- DATA_ISSUE：0
- UNKNOWN_REASON：1

第二阶段结论：
- UNKNOWN_CHANGE 绝大多数可解释为“结构拆分”，不是纯噪声。
- 货量有触发作用，但“结构重分配”是主因。

第三阶段：线路负载弹性区间（Slack）与阈值验证
==================================================

一、目标
在“结构复用 + 结构拆分”规律上，提取每条线路负载弹性区间，并验证变化触发阈值。

二、EXACT_COPY 数据校验（先行）
1) EXACT_COPY 样本：5127
2) |delta| 分布：
- p50=0.0647, p75=0.1386, p90=0.2600, p99=0.6431, max=8.0288
3) 异常识别：
- |delta|>p99：52 条
- |delta|>3*p50：812 条（阈值偏敏感）

说明：为避免异常污染，slack 提取采用“剔除 |delta|>p99”后的 EXACT_COPY 样本。

三、极端样本核查（门店级）
1) 2025-11-13 vs 2025-11-12, route 112
- 同店铺集合下，store_id=394148 单店负载 0.0360 -> 7.9338
- 线路 delta=+8.0288

2) 2026-04-03 vs 2026-04-02, route 140
- 多店同步显著下降，delta=-2.5806

3) 2026-03-21 vs 2026-03-20, route 777
- 三店均大幅下降，delta=-1.2257

判定：
- 存在 DATA_ISSUE_CANDIDATE（疑似单位/录入/聚合异常或口径变动）。
- 异常样本已从 slack 统计口径中排除。

四、线路弹性区间提取（EXACT_COPY 且通过校验）
定义：
- 稳定负载区间：p10 ~ p90
- 极限区间：min ~ max

按组差异（线路均值）：
1) route_group
- W1：平均稳定区间宽度 0.6566
- W2：0.5209
- OTHER：0.7363

2) route_type
- 市区：0.5497
- 地铁：0.2397（最窄）
- 郊区周边：1.0575
- 周边城市：1.0369
- 机场：0.8497
- 高铁站：0.7363

3) is_city
- city：0.5497
- non_city_or_null：0.7491

五、触发阈值验证（PARTIAL_CHANGE / UNKNOWN_CHANGE）
样本数：1683

1) 超出历史弹性区间（<p10 或 >p90）
- 481 条，比例 28.58%

2) 超过 normal_load_limit
- 46 条，比例 2.73%

解释：
- 结构变化并非都由“超上限”触发。
- 但“超出该线路历史弹性区间”与结构变化关联明显。

六、对照样例结论
1) 区间内但发生变化：存在
- 说明结构调整还有非负载因素（分配策略、跨线平衡等）

2) 刚超过区间就变化：存在
- 例如 route 201：route_load 比 p90 仅高 0.0014 即发生 PARTIAL_CHANGE
- 说明 p90 可作为较敏感的预警拐点

==================================================
总回答（面向业务问题）
==================================================

问题1：每条线路在什么负载范围内可保持结构不变？
- 建议以“该 route_id 在通过校验的 EXACT_COPY 样本中的 p10~p90”作为稳定区间。

问题2：超过哪个点开始发生结构变化？
- 经验上，接近并超过该线路历史 p90 后，变化概率明显上升。
- 对市区线，超过 normal_load_limit 时变化率更高，但覆盖范围小于“超历史弹性区间”。

第四阶段：人工调度历史相似日匹配（只读）
==================================================

一、分析目标
在同一 route_id + 同一 day_type 维度下，判断当前线路货量是否可在历史同类日中找到“几乎一致（±5%）”参考日。

二、route_load 计算口径（主口径）
route_load = Σ(
  ambient_turnover_boxes/207
+ ambient_cartons/380
+ ambient_full_cases/380
+ frozen_bags/120
+ frozen_cartons/380
+ cold_bins/350
+ cold_cartons/380
)

说明：
- 从 human_dispatch_routes 聚合按 delivery_date + day_type + route_id 计算。
- day_type 由 chaos_replay_tasks（calc_version='chaos_replay_task_v1'）补齐。
- 相似判定主阈值：±5%。

三、相似判定规则
同一 route_id、同一 day_type 下：
abs(current_route_load - historical_route_load) / nullif(historical_route_load,0) <= 0.05

特殊处理：
- historical_route_load = 0 且 current_route_load = 0：视为相似
- historical_route_load = 0 且 current_route_load != 0：不相似

四、全量汇总（5%）

第五阶段：负载分布位置 → 结构变化概率曲线（只读）
==================================================

一、目标
从固定阈值判断，提升为“当前负载在本线路历史分布中的位置”对应的结构变化概率曲线。

二、基础口径
1) 数据源
- human_dispatch_routes：按 delivery_date + day_type + route_id 聚合 route_load
- chaos_replay_tasks：提供 day_type
- rengong_route_distance_load_profiles：提供 route_group / route_type / is_city / normal_load_limit

2) reuse_type（沿用前序）
- EXACT_COPY
- SAME_STORES_DIFF_ORDER
- PARTIAL_CHANGE
- UNKNOWN_CHANGE
- NEW_ROUTE
- MISSING_PREV_ROUTE

3) 结构变化定义
- 变化：SAME_STORES_DIFF_ORDER / PARTIAL_CHANGE / UNKNOWN_CHANGE / NEW_ROUTE / MISSING_PREV_ROUTE
- 不变化：EXACT_COPY

4) DATA_ISSUE_CANDIDATE 剔除
- 在 EXACT_COPY 中，若 |delta| > p99 或 |delta| > 3*p50 则标记为候选异常并剔除。
- 本次候选异常：812 条（从 7410 总线路日样本中剔除）。

三、历史分布构造与 LOW_SAMPLE
按 route_id + day_type 计算：
- p10, p25, p50, p75, p90, min, max, sample_count

统计：
- route_id+day_type 组合总数：98
- LOW_SAMPLE（sample_count < 15）：7 组
- 主结论仅使用 sample_count >= 15 的组合。

四、负载位置字段
对每条线路日计算：
- load_vs_p50 = current_route_load / p50
- load_vs_p90 = current_route_load / p90
- load_percentile_band：
  BELOW_P10 / P10_P25 / P25_P50 / P50_P75 / P75_P90 / ABOVE_P90
- limit_ratio = current_route_load / normal_load_limit（无上限记为 NO_LIMIT）

五、全量概率曲线（sample_count >=15）
1) 按 load_percentile_band
- BELOW_P10：525，变化率 35.43%
- P10_P25：989，变化率 24.37%
- P25_P50：1660，变化率 22.47%
- P50_P75：1627，变化率 35.34%
- P75_P90：979，变化率 48.72%
- ABOVE_P90：790，变化率 52.78%

2) 按 load_vs_p90 分桶
- <=0.6：4156，变化率 29.09%
- 0.6~0.8：1056，变化率 42.52%
- 0.8~0.9：294，变化率 34.69%
- 0.9~1.0：366，变化率 35.79%
- 1.0~1.1：253，变化率 35.57%
- 1.1~1.3：258，变化率 55.04%
- >1.3：187，变化率 78.07%

六、分层曲线
1) day_type（load_vs_p90 分桶）
- NORMAL：>1.3 桶变化率 78.49%
- COLD_ONLY：>1.3 桶变化率 73.33%

2) route_group
- W1：>1.3 桶 82.91%
- W2：>1.3 桶 78.69%
- OTHER：整体低，>1.3 桶 11.11%（样本少）

3) route_type（主要类型）
- 市区：>1.3 桶 82.96%
- 周边城市：>1.3 桶 76.47%
- 地铁：>1.3 桶 77.27%
- 郊区周边：>1.3 桶 66.67%（样本少）
- 机场：>1.3 桶 100%（n=1）
- 高铁站：整体低波动、高一致性

4) is_city
- city：>1.3 桶 82.96%
- non_city_or_null：>1.3 桶 65.38%

七、关键行为区间识别
1) 变化率明显上升区间
- 从 P75_P90 开始明显上升（48.72%）
- ABOVE_P90 更高（52.78%）

2) ABOVE_P90 是否高于 P50_P75、P75_P90
- 是：52.78% > 35.34% > 48.72%

3) load_vs_p90 > 1.0 后是否显著升高
- 在 1.0~1.1 还不显著（35.57%）
- 到 1.1~1.3 明显上升（55.04%）
- >1.3 非常显著（78.07%）

4) normal_load_limit 解释力 vs 历史 p90
- 历史 p90：覆盖广（全量可算），>p90 的变化率 54.15%，<=p90 为 32.20%
- normal_load_limit：解释力强但覆盖窄（仅有上限口径的 1866 样本，且 >=1 仅 68 样本）
  - >=1 变化率 88.24%
  - <1 变化率 36.76%
结论：
- 对“强风险信号”而言，normal_load_limit>=1 很强；
- 对“日常连续预警”而言，历史 p90 更稳健、覆盖更全面。

九、最终回答
“人工调度的结构变化概率，是否随着 route_load 接近或超过该线路历史 p90 而明显上升？”

答：是。
- 从 P75_P90 开始上升，ABOVE_P90 继续上升；
- 当 load_vs_p90 > 1.1 后，上升显著加速；
- >1.3 区间变化概率最高（约 78%）。
